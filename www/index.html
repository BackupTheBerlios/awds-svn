What is AWDS about?

AWDS (Ad-hoc Wireless Distribution Service) is a Layer 2 routing protocol for wireless mesh networks. It provides transparent Ethernet-like access to all participating nodes, thus easily allowing the employment of different higher level protocols like IP (with DHCP), IPv6, AppleTalk, ...
A Linux implementation of AWDS can be downloaded from our download section. Ports to other platforms are highly appreciated!
AWDS is a development effort at the Real-Time Systems and Communication working group of the Otto-von-Guericke University, Magdeburg.

2008-10-14 - AWDS 8.8 and GEA 3.3 released

After testing AWDS in a real test bed at the Magdeburger Open Source Tag (MOST) we are proud to release the new version 8.8 of AWDS and the corresponding GEA 3.3. Some changes:
  • Faster detection of broken links
  • Use of ETX metric as default
  • IPv6 support for the TopoViewer connection
  • Some bugfixes for GCC-4.4 and GCC-3.3

2008-10-14 - AWDS developers use Git now

When AWDS started, we used Subversion for managing the source code. However, when more developers joined us, we had to learn about Subversions deficits. Git was chosen as alternative. There was a slow and careful migration process, but now, the latest version of AWDS and its helper library GEA can only be found in the corresponding Git repositories.

2007-03-29 - AWDS 6.0 released

A new major release of AWDS is now available. It includes changes in the protocol header, which makes it incompatible with prior releases. This was necessary for including a great new feature: route tracing. It allows to track the route a packet takes. The feature is included in the ping command. The -p option allows ping to not only check if a node is reachable, but also makes it show the path the packets take.
